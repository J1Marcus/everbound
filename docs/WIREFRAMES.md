## Document Purpose

This document provides wireframes for the Digital Memoir Platform's key user interfaces and workflows. These wireframes are based on the requirements defined in [`PROJECT_SCOPE.md`](PROJECT_SCOPE.md) and [`USER_WORKFLOWS.md`](USER_WORKFLOWS.md).

---

## Design Principles

### 1. Print-First Interface
- Every screen reinforces the goal: creating a physical book
- Progress indicators show path to completion
- Visual emphasis on narrative quality over speed

### 2. Simplicity & Focus
- Minimal distractions
- Clear calls-to-action
- One primary task per screen
- No social features or gamification

### 3. Voice Preservation
- Interface adapts to user, not vice versa
- Natural language prompts
- Conversational tone
- Respect for narrator's authority

### 4. Quality Enforcement
- Transparent quality requirements
- Clear feedback when standards not met
- Helpful guidance, not frustration
- Block bad outcomes, don't hide them

### Typography (Suggested)

```
Headings:
- Font: Merriweather or Georgia (serif, literary feel)
- Sizes: H1: 32px, H2: 24px, H3: 20px

Body Text:
- Font: Open Sans or system font (readable, clean)
- Size: 16px
- Line height: 1.6

Interface Elements:
- Font: System font (native feel)
- Size: 14-16px
```

### Spacing & Layout

```
Container Width: 1200px max
Content Width: 800px max (for reading)
Padding: 24px standard, 48px large sections
Border Radius: 8px (cards), 4px (buttons)
```

### Interactive Elements

```
Buttons:
- Primary: Solid color, prominent
- Secondary: Outline style
- Tertiary: Text only
- Minimum touch target: 44x44px

Forms:
- Input height: 48px
- Label above input
- Clear error states
- Helpful placeholder text

Cards:
- Subtle shadow
- Hover state for clickable cards
- Clear visual hierarchy
```

## Mobile Considerations

### Responsive Breakpoints

```
Mobile: < 768px
Tablet: 768px - 1024px
Desktop: > 1024px
```

### Mobile-Specific Adaptations

1. **Navigation**
   - Hamburger menu for main navigation
   - Bottom tab bar for primary sections
   - Swipe gestures for chapter navigation

2. **Memory Capture**
   - Voice recording prioritized on mobile
   - Camera integration for photo memories
   - Simplified forms with fewer fields visible

3. **Chapter Review**
   - Scrollable chapter view
   - Sticky action buttons at bottom
   - Simplified quality metrics

4. **Progress Indicators**
   - Compact progress bars
   - Collapsible sections
   - Focus on next action

## Accessibility Considerations

### WCAG 2.1 AA Compliance

1. **Color Contrast**
   - Minimum 4.5:1 for body text
   - Minimum 3:1 for large text
   - Don't rely on color alone for meaning

2. **Keyboard Navigation**
   - All interactive elements keyboard accessible
   - Visible focus indicators
   - Logical tab order

3. **Screen Readers**
   - Semantic HTML
   - ARIA labels where needed
   - Alt text for all images
   - Status announcements for dynamic content

4. **Text Scaling**
   - Support up to 200% zoom
   - Responsive text sizing
   - No horizontal scrolling
